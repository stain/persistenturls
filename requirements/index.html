PURL 2 Project Requirements

 

Security

  • Common login

Account Creation and Maintenance

  • Register a new user
  • Modify user information
  • Search user information
  • Delete a user1

PURL Creation and Maintenance

  • Create a PURL
  • Create a partial redirect PURL
  • Modify a PURL
  • Chain a PURL
  • Clone a PURL
  • Batch add PURLs
  • Batch modify PURLs
  • PURL Validation
  • Type a PURL (this reflects the W3C HTTP header change recommendations for giving URLs to non-networked resources)
  • Search PURL metadata
  • Delete PURLs1

Group Creation and Maintenance

  • Create a new group
  • Modify group information
  • Search group information
  • Delete a group1

Domain Creation and Maintenance

  • Request a top-level domain
  • Create a subdomain in an existing domain
  • Modify existing domain information
  • Search domain information
  • Delete a domain1

Search

  • Search PURL metadata
  • Search user information
  • Search group information
  • Search domain information
  • Search tomb-stoned PURL metadata
  • Search tomb-stoned user information
  • Search tomb-stoned group information
  • Search tomb-stoned domain information

See also the Public URL design.

Notes

1. (Zepheira suggests tombstone rather than delete as we believe it will be helpful to keep track and search deleted groups to determine future spam / phishing patterns).
